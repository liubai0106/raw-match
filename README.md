## 一、应用场景

### 典型工作流

```
拍摄 → 相机同时输出 JPG + RAW → 选片 → 找出对应的 RAW → 精修
```

### 痛点

- 一次拍摄几百张，相机同时保存 JPG + ARW(RAW)
- 从中选了 10 张 JPG 要精修
- 需要找到这 10 张 JPG 对应的 RAW 原文件
- 手动逐个查找太慢且容易遗漏

### 解决方案

**文件匹配**：拖入左侧(选片)，拖入右侧(全部原图)，一键导出匹配的 RAW。

---

## 二、核心功能

```
┌─────────────────────────────────────────────────────┐
│  左侧面板                  右侧面板                  │
│  (选片图片)               (全部 JPG + RAW)          │
│  1.jpg  ─────┐                                       │
│  5.jpg  ─────┼──→ 文件名匹配 ──→ 导出匹配文件        │
│ 12.jpg  ─────┘                                       │
└─────────────────────────────────────────────────────┘
```

**匹配逻辑**：按文件主名(stem)匹配。左侧 `1.jpg` 匹配到右侧 `1.arw` + `1.jpg`，两个都导出。

## 四、使用指南

### 操作步骤

```
1. 双击打开 exe

2. 左侧拖入：选片文件夹(只有 JPG 的那个)
   → 自动识别 23 种图片格式

3. 右侧拖入：全部原图文件夹(JPG + RAW 都在)
   → 自动识别 30 种 RAW + 22 种图片

4. 点击 [一键导出所有匹配文件]

5. 选择导出目录 → 完成
```

### 示例

```
左侧(选片)：           右侧(全部)：           导出结果：
├── 1.jpg             ├── 1.jpg             ├── 1.jpg
├── 5.jpg             ├── 1.arw             ├── 1.arw
└── 12.jpg            ├── 2.jpg             ├── 5.jpg
                      ├── 2.arw             ├── 5.arw
                      ├── 3.jpg             ├── 12.jpg
                      ├── 3.arw             └── 12.arw
                      ├── 5.jpg
                      ├── 5.arw
                      ├── 12.jpg
                      └── 12.arw
```

## 五、版本对比: 轻量版 vs 校验版

### 快速对比

| 对比项   |       轻量版        |           校验版         |
| -------- | -------------------- | -------------------------|
| 文件     | 文件匹配.exe (10 MB) | 文件匹配终版.exe (61 MB)  |
| 源码     | 文件匹配.py          | 文件匹配终版.py           |
| 匹配方式 | 文件名匹配           | 文件名 + **图像指纹校验** |
| 防篡改   | ❌ 无                 | ✅ 有(pHash 感知哈希)    |

> 两者支持格式相同：左侧 23 种图片，右侧 30 种 RAW + 22 种图片

### 什么时候选 轻量版?

- 右侧文件夹是你自己拍的，RAW 都是真的
- 追求轻量，方便分享给朋友
- 快速操作，不需要二次校验
- 文件多(几千张)，速度优先

### 什么时候选 校验版?

- 右侧文件夹来源不可信(别人拷来的、二手硬盘)
- 有人可能改过 RAW 文件名(比如 2.arw 改成 1.arw)
- 宁可漏掉，也不能匹配错
- 对准确性要求极高

### 区别演示

```
场景：右侧 1.arw 被人改过名(实际是 2.arw)

轻量版 (文件名匹配)：
  左 1.jpg → 匹配 → 导出 1.arw ✗(错的！导出的是别人的 RAW)

校验版 (pHash 校验)：
  左 1.jpg → 匹配 → pHash 比对 → 距离=136(>10) → 跳过 ✓(不导出)
```

### 建议

**日常使用 轻量版，重要项目 校验版 二次校验。**

## 七、技术原理

### 轻量版 核心原理：文件名 Stem 匹配

```
文件：DSC00001.JPG
      ─────── ────
      stem    ext

stem = 文件名去掉扩展名后的部分
```

**匹配流程**：

```
1. 遍历左侧文件夹，提取所有图片文件的 stem
   → base_names = {"DSC00001", "DSC00005", "DSC00012"}

2. 遍历右侧文件夹，按 stem 分组存储
   → match_files = {
       "DSC00001": {"jpg": "D:/xxx/DSC00001.JPG", "arw": "D:/xxx/DSC00001.ARW"},
       "DSC00002": {"jpg": "...", "arw": "..."},
       ...
     }

3. 导出：对每个 base_name，如果在 match_files 中存在，复制所有右侧文件
```

**时间复杂度**：O(n + m)，n 和 m 分别是左右侧文件数。

### 校验版 增强原理：pHash 感知哈希

v3 在文件名匹配基础上，增加**图像内容校验**。

#### pHash 算法流程

```
原始图片
   ↓
1. 缩放到 32×32 小图(去除高频细节)
   ↓
2. 转灰度图
   ↓
3. 2D DCT 离散余弦变换(提取频率特征)
   ↓
4. 取左上角 8×8 低频系数(最能代表图像内容)
   ↓
5. 计算 64 个系数的平均值
   ↓
6. 每个系数 > 平均值 → 1，否则 → 0
   ↓
7. 拼接成 64 位整数 = pHash 值
```

#### 比对方法

```
图片 A → pHash → 0b10110011...
图片 B → pHash → 0b10110100...

汉明距离 = 两个 hash 值不同位的数量
         = XOR 后数 1 的个数

距离 = 0   → 同一张图
距离 ≤ 10 → 匹配(容忍轻微压缩/缩放)
距离 > 10 → 不同图片
```

#### RAW 文件的处理

RAW 文件无法直接打开，但**多数相机 RAW 内嵌一张预览 JPG**(就是你在相机屏幕上看到的那张)。

```
RAW 文件
   ↓
rawpy.extract_thumb()  ← 提取嵌入的预览 JPG
   ↓
PIL.Image.open()       ← 作为普通图片打开
   ↓
pHash 比对              ← 和左侧 JPG 比对
```

**关键发现**：同一张照片的相机直出 JPG 和 RAW 嵌入预览，**几乎字节级一致**，pHash 距离通常为 0。

#### 防篡改原理

```
场景：有人把 DSC00002.ARW 改名为 DSC00001.ARW

v4： DSC00001.JPG → stem 匹配 → 导出 DSC00001.ARW(假的)  ✗

v3： DSC00001.JPG → stem 匹配 → 提取 DSC00001.ARW 预览
     → pHash(左 JPG) vs pHash(右 RAW 预览)
     → 距离 = 136(>10)
     → 判定：不是同一张图 → 跳过                    ✓
```

### 两个版本的技术差异

| 技术点   |      轻量版       |              校验版               |
| -------- | ----------------- | ---------------------------------- |
| 匹配算法 | 字符串 stem 比对  | stem 比对 + pHash                  |
| 图像库   | 无                | Pillow + rawpy + imagehash + scipy |
| 哈希算法 | 无                | pHash(基于 DCT)                    |
| 阈值判定 | 无                | 汉明距离 ≤ 10                      |
| 依赖数量 | 1 个(tkinterdnd2) | 5 个                               |
| 体积     | 10 MB             | 61 MB                              |
